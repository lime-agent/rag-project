# 기술 스택 결정 기록

## 임베딩 모델: nomic-embed-text (Ollama)

**결정**: Ollama의 `nomic-embed-text` 사용

**이유**:
- Ollama를 이미 LLM 용도로 쓰고 있어서 추가 의존성 없이 한 줄로 설치 가능
- `ollama pull nomic-embed-text` — 별도 Python 패키지 불필요
- 768차원 벡터, L2 정규화 (코사인 유사도 최적화)
- 다국어 지원 (한국어 포함)

**트레이드오프**: Ollama 서버가 실행 중이어야 함. 서버 없이 쓰려면 FastEmbed 같은 대안 존재.

## 벡터 DB: ChromaDB

**결정**: ChromaDB 로컬 파일 모드 사용

**이유**:
- 서버 없이 로컬 파일(`.chroma/`)에 저장
- `pip install chromadb` 한 줄로 설치
- Python API 단순함

**비교 검토**:
- FAISS: 설치 복잡, 우리 규모에서 성능 차이 없음
- Pinecone: 유료, 외부 서버 필요
- Weaviate: 설정 복잡

## LLM: Ollama (qwen2.5:3b)

**결정**: Ollama로 로컬 LLM 실행

**이유**:
- 외부 API 키 불필요
- 사내망에서도 동작
- qwen2.5:3b는 3B 파라미터로 속도·품질 균형

**트레이드오프**: Claude API 대비 답변 품질 낮음. 6회차에서 LiteLLM을 통해 다른 모델로 교체 가능.
